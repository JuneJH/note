# 动态规划

> 动态规划(Dynamic Programming)即DP，是解决复杂问题的一种思路，其中最核心的思想就是将问题分解成子问题，只要子问题被解决了，复杂问题也就被解决了
>
> 和分治法的区别在于: 核心区别在于子问题是否存在重叠性，相同的子问题可能会被多次计算，动态规划通过存储子问题的结果（记忆化搜索或自顶向上计算）来避免重复计算，相比之下，分治法通常将问题拆分成相互独立的子问题，然后合并子问题的结果

### 1. 最优子结构

1. 最优子结构指的是：某个问题的最优解可以通过其他子问题的最优解来构造，比如需要求解P问题，那么把P问题拆分为若干个子问题P1、P2、P3...求的他们的最优解，再根据子问题的最优解组合出P的最优解。
2. 最优子结构是许多算法设计策略(分治法、贪心算法、动态规划)的一个重要前提。对于DP来说，没有最优子结构就无法通过拆分子问题来推导原问题的解

### 2. 重叠子问题

1. 在拆分子问题的过程中，会出现重复的子问题求解
2. 在分治法中虽然也会把问题拆分成子问题，但很多分治问题里，这些子问题只是少量重叠

### 3. 存储子问题的解

1. 因为存在大量重复子问题，因此需要缓存
2. 记忆化搜索（自上而下）
3. 表格法（自下而上）
4. 正因存子问题的解，从而可以区分分治法的区别，纯纯的分治法不需刻意缓存子问题的解，动态可以理解为沿着子问题规模的增加而得到新的解并维护和更新

### 4. 状态转移方程

1. 状态转移方程就是描述从一个状态如何过渡到另一状态的公式，本质上是一种递推/递归关系，指导我们如何从小规模子问题的解一步步累积得到大规模的解
2. 一个问题的解可以由它的子问题解来合成，状态转移方程就是把这种合成过程用正式的数学或伪代码表达出来
